Q: Will this break existing questions?
A: NO. New fields are optional. Existing questions continue to work.

Q: Do I need to change the frontend?
A: NO. Frontend stays exactly the same. All backend.

Q: How is email sending non-blocking?
A: Answer is saved immediately. Email is sent asynchronously in background.

Q: What happens if email fails?
A: Failure is logged. Status tracked in database (emailError field).
   Mentor still gets successful response.

Q: Can students see each other's phone numbers?
A: NO. Phone numbers are NEVER exposed in public APIs.
   Only mentors/admins can see them.

Q: What if student provides fake phone number?
A: Backend rejects with specific error message.
   Student must provide valid number to submit question.
